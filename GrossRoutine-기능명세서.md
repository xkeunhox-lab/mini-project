# GrossRoutine 기능 명세서

## 1. 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | GrossRoutine — 루틴 기반 운동 기록 (구 "Lift Logbook"에서 리브랜딩) |
| 타겟 사용자 | 기존 운동 루틴과 기록 습관이 있는 사용자 (A-세그먼트) |
| 핵심 가설 | 새로 처음부터 기록하게 만드는 것보다, **루틴을 빠르게 불러와 기록하고 페인포인트를 해결**해주면 재사용률이 높다 |
| 구현 방식 | 단일 HTML 파일 (CDN React 없이 Vanilla JS), `localStorage` 기반 데이터 저장 |
| 파일 위치 | `lift-logbook.html` (파일명은 유지, 서비스명만 GrossRoutine으로 변경) |
| 비주얼 컨셉 | 다크 테마 + 브랜드 그린 액센트. 상세는 "7. 비주얼 아이덴티티 / 디자인 시스템" 참고 |

## 2. 정보 구조 (IA)

앱은 2개의 탭으로 구성된다.

- **📋 루틴 관리 탭**: 재사용할 운동 루틴을 등록·수정·삭제
- **📅 기록 탭**: 캘린더에서 날짜를 선택해 실제 운동을 기록

## 3. 데이터 모델

```
routines: [
  {
    id: string,
    routineName: string,
    exercises: [
      { exerciseName: string, sets: [{ weight: number, reps: number }] }
    ],
    createdDate: string
  }
]

records: {
  [dateStr]: {
    routineId: string | null,   // 이 날짜에 불러온 루틴 (없으면 null)
    exercises: [
      { exerciseName: string, sets: [{ weight: number, reps: number, completed: boolean }] }
    ],
    memo: string
  }
}
```

두 데이터 모두 `localStorage`에 각각 `routines`, `records` 키로 저장된다.

`completed`(세트 완료 체크)는 **그날의 실제 수행 여부**를 나타내는 값이라 루틴 정의(`routines`)에는 의미가 없다. 루틴을 불러오거나 과거 기록을 불러올 때는 무게/횟수만 가져오고 `completed`는 항상 `false`로 초기화된다 (전날 체크한 상태가 오늘 화면에 그대로 남아있지 않도록).

루틴은 생성 순서(index)에 따라 6가지 색상(`color-0` ~ `color-5`)이 순환 배정되며, 이 색상은 루틴 카드·루틴 불러오기 목록·캘린더 배지에서 동일하게 사용되어 어떤 루틴인지 한눈에 구분할 수 있다.

## 4. 기능 명세 — 루틴 관리 탭

### 4.1 루틴 목록
- 등록된 루틴을 카드 형태로 표시. 카드 배경은 루틴별 고유 색상(그라데이션)
- 각 카드에는 루틴명과 포함된 운동 종목 목록이 미리보기로 표시
- "➕ 루틴 추가" 카드를 클릭하면 루틴 생성 모달이 열림

### 4.2 루틴 상세/수정 흐름
- 루틴 카드 클릭 → **루틴 상세 팝업**에 운동 종목별 세트 정보 표시
- 상세 팝업의 **운동 항목을 클릭하면 중간 단계 없이 바로 수정 모드로 진입** (기존 "편집" 버튼 클릭 → 상세보기 → 수정의 2단계 흐름을 1단계로 단순화)
- 상세 팝업 하단에 "🗑️ 삭제" 버튼으로 루틴 전체 삭제 가능 (확인창 필수)

### 4.3 운동 종목 추가/수정 폼
- 운동 종목명(자동완성 지원), 세트 수, 세트별 무게(kg)/횟수(회) 입력
- 세트 수를 늘리거나 줄이면 기존에 입력해둔 값은 유지된 채로 입력창 개수만 조정됨 (입력값 유실 없음)
- 운동 종목 삭제 시 확인창 표시
- 운동명 입력 시 기존에 등록된 모든 운동(루틴 + 기록 데이터 전체)에서 자동완성 후보를 제공

### 4.4 루틴 저장
- 루틴명과 최소 1개 이상의 운동 종목이 있어야 저장 가능
- 신규 생성 / 기존 루틴 수정 모두 동일한 모달 사용

## 5. 기능 명세 — 기록 탭 (캘린더)

### 5.1 캘린더 뷰
- 월 단위 캘린더, 이전/다음 달 이동 가능
- 기록이 저장된 날짜에는 ✓ 표시
- 해당 날짜에 루틴을 불러와 저장한 경우, **루틴명 배지가 루틴 고유 색상으로 표시**되어 어떤 루틴을 한 날인지 캘린더에서 바로 식별 가능
- 루틴명 배지 옆 **✕ 아이콘 클릭 시 확인창 후 해당 날짜의 기록 전체 삭제** (캘린더 화면에서 날짜 진입 없이 바로 삭제)
- 기록이 있는 날짜 하단에 **그날의 총 볼륨**을 함께 표시 ("총 120kg" 형태). 총 볼륨은 그날 기록한 **모든 세트의 무게를 합산**한 값 (세트별 횟수는 곱하지 않음)

### 5.2 날짜 선택 → 임시 편집본(Draft) 진입
- 날짜를 클릭하면 해당 날짜의 저장된 기록(없으면 빈 기록)을 **복사한 임시 편집본**으로 진입
- 화면에서 이루어지는 모든 조작(운동 추가/삭제, 세트 값 수정, 세트 수 변경, 루틴 불러오기, 메모 작성)은 **임시 편집본에만 반영**되고 실제 저장 데이터에는 영향을 주지 않음
- **"💾 저장" 버튼을 눌러야만** 임시 편집본이 실제 기록으로 커밋되어 `localStorage`에 반영되고 캘린더에도 표시됨
- **"← 돌아가기" 버튼**을 누르면 임시 편집본은 그대로 폐기되어 원래 저장된 상태가 유지됨 (실수로 되돌릴 수 없는 저장 방지)

### 5.3 루틴 불러오기
- "📂 루틴 불러오기" 버튼 클릭 → 등록된 루틴 목록(색상 카드)이 표시됨
- 루틴을 선택하면 **해당 루틴의 모든 운동 종목이 한 번에 "오늘의 운동" 섹션에 표시**됨 (개별 운동을 하나씩 클릭할 필요 없음)
- **운동 종목 목록은 항상 루틴의 현재 정의를 기준으로 표시**된다. 이후 루틴에 운동이 추가/삭제되면 다음 불러오기부터 바로 반영됨
- 각 운동의 **무게/횟수 값**은, 동일 루틴으로 과거에 기록한 이력이 있고 그 기록에 같은 운동이 있으면 **가장 최근 기록값**을 우선 사용하고, 없으면(루틴에 새로 추가된 운동 등) 루틴에 정의된 기본값을 사용 — 점진적 과부하 추적과 최신 루틴 반영이 동시에 가능

### 5.4 이전 기록 불러오기
- "🕐 이전 기록 불러오기" 버튼 클릭 → **실제로 기록이 존재하는 과거 날짜 목록**(최신순)이 표시됨. 오늘 날짜는 목록에서 제외
- 각 항목에는 날짜, (루틴을 불러와 기록한 날이면) 루틴 배지, 그날 기록한 운동명 목록이 미리보기로 표시됨
- 루틴 불러오기와 달리 **루틴 연결 여부와 무관하게 실제로 기록했던 모든 날짜**를 대상으로 하며, 하루를 선택하면 그날의 운동·세트·무게·횟수를 통째로 오늘의 운동으로 복사
- 세트 **완료 체크는 항상 초기화**됨 (과거 기록의 완료 여부가 오늘로 넘어오지 않도록)
- 루틴 불러오기가 "정해진 루틴대로" 기록하는 기능이라면, 이 기능은 **"직전 컨디션을 참고해 오늘 컨디션에 맞게 조정"**하기 위한 기능 — 루틴 유무와 상관없이 사용 가능

### 5.5 오늘의 운동 — 개별 수정
- 각 운동 항목에서 세트별 무게(kg)/횟수(회)를 직접 입력·수정 가능
- 각 세트 줄 맨 앞의 **체크박스**로 그 세트를 완료했는지 표시 가능
- 각 세트 줄 끝의 **"−" 버튼**으로 해당 세트만 개별 삭제 (최소 1개 세트는 항상 유지되도록 방어 — 마지막 1개를 지우려 하면 안내 후 취소)
- 운동 항목 하단의 **"+ 세트 추가" 버튼**으로 세트를 추가 (직전 세트의 무게/횟수 값을 기본값으로 복사)
- 운동 항목 삭제 시 확인창 표시

### 5.6 운동 추가 (신규 종목)
- "➕ 운동 추가" 버튼 클릭 → 루틴 관리와 동일한 형태의 입력 폼(운동명 자동완성 + 세트 수 + 세트별 무게/횟수) 표시
- "추가" 버튼 클릭 시:
  - 해당 날짜에 불러온 루틴이 있는 경우 → **"루틴에도 추가할까요?"** 확인창 표시
    - **예** → 방금 추가한 운동이 원본 루틴에도 추가되어 저장되고 "루틴에 추가되었습니다" 알림 표시 (다음부터 이 루틴을 불러오면 항상 포함됨, 5.3 참고)
    - **아니오** → 오늘의 운동에만 추가되고 루틴 원본은 변경되지 않음
  - 불러온 루틴이 없는 날짜(수동 기록)는 확인창 없이 오늘의 운동에만 추가

### 5.7 메모
- 자유 텍스트로 오늘의 운동 컨디션 등을 기록 가능 (저장 버튼을 눌러야 반영)

### 5.8 저장
- "💾 저장" 버튼 클릭 시 임시 편집본(운동 목록 + 메모)을 실제 기록으로 커밋하고 캘린더 화면으로 복귀
- **오늘의 운동이 1개도 없으면 저장되지 않는다.** "오늘의 운동을 1개 이상 추가해주세요" 안내 후 저장을 중단하고 화면에 머무름 (빈 기록이 캘린더에 남는 것을 방지)

## 6. 주요 UX 원칙 (개발 과정에서 확정된 설계 원칙)

1. **클릭 단계 최소화**: 상세보기 전용 화면을 거치지 않고, 항목을 클릭하면 바로 수정 가능한 상태로 진입
2. **저장은 명시적으로만 일어난다**: 입력 중 변경사항이 실시간으로 영구 저장되지 않고, 반드시 "저장" 버튼을 눌러야 반영됨 → "돌아가기"로 안전하게 이탈 가능
3. **파괴적 행동에는 항상 확인창**: 루틴 삭제, 운동 종목 삭제, 날짜 기록 삭제 등 되돌릴 수 없는 작업 전 confirm 필수
4. **색상으로 루틴 식별**: 루틴 관리, 루틴 불러오기, 캘린더 배지에서 동일한 색상 체계를 사용해 어떤 루틴인지 시각적으로 즉시 구분
5. **최근 기록 우선 로드**: 루틴을 불러올 때, 이미 있는 운동은 루틴 원본 값이 아니라 가장 최근 기록값을 기본값으로 제시해 매번 무게를 처음부터 다시 입력하지 않도록 함
6. **의미 없는 저장은 막는다**: 운동을 하나도 추가하지 않은 빈 기록은 저장 자체가 되지 않도록 막아, 캘린더에 빈 ✓ 표시가 쌓이는 것을 예방
7. **루틴은 항상 최신 정의를 따른다**: "최근 기록 우선 로드"는 무게/횟수 같은 *값*에만 적용되고, 운동 *목록*(어떤 종목이 포함되는지)은 항상 루틴의 현재 정의를 기준으로 한다. 그래야 기록 탭에서 루틴에 새 운동을 추가한 뒤 루틴을 다시 불러왔을 때 그 운동이 누락되지 않음

## 7. 비주얼 아이덴티티 / 디자인 시스템

### 7.1 브랜드
- 서비스명 **GrossRoutine** — 헤더에 헤비 웨이트 워드마크로 노출 (다크 배경 위 브랜드 컬러 + 은은한 글로우)
- 헤더 구조: 브랜드 워드마크 → 서브타이틀("루틴 기반 운동 기록") → 태그라인("편리하게 기록하고 효율적으로 관리하기") 3단 스택

### 7.2 컬러 시스템
| 토큰 | 값 | 용도 |
|---|---|---|
| `--primary` (브랜드 컬러) | `#18DB4E` | 브랜드 워드마크, 주요 버튼(저장 등), 활성 탭, 포커스 링, 오늘 날짜 강조, 루틴 1번 색상 |
| `--primary-dark` | `#13A93C` | primary 버튼 hover/pressed |
| `--success` | `#18DB4E` (primary와 동일) | 저장·불러오기 등 긍정적 액션 버튼 — 브랜드 컬러와 통일해 사용 |
| `--danger` | `#FF5A52` | 삭제 등 파괴적 액션 |
| `--info` | `#2F86EB` | 정보성 보조 액션 |
| 배경/서페이스 | `#0B0D08`(페이지) / `#171B10`(카드·모달) | 다크 테마 기본 배경 |
| 텍스트 | `#F4F6EE`(주요) / `#98A187`(보조) | 다크 배경 위 텍스트 |

- 밝은 그린(`--primary`, `--success`) 배경 위에는 흰 텍스트 대신 **다크 텍스트**를 사용해 명도 대비를 확보 (예: 저장 버튼, 루틴 1번 카드, 캘린더 완료 배지)
- 루틴은 생성 순서에 따라 6가지 컬러가 순환 배정됨: 그린(브랜드) → 블루 → 마젠타 → 틸 → 오렌지 → 바이올렛. 각 색은 그라데이션 카드로 표현되며 루틴 카드 · 루틴 불러오기 목록 · 캘린더 배지에서 동일하게 사용

### 7.3 다크 테마 적용 범위
- 전체 화면(헤더, 탭, 카드, 모달, 캘린더, 입력 폼)에 다크 서페이스 적용
- 캘린더의 "오늘" 날짜는 브랜드 컬러 테두리 + 옅은 틴트로 강조, 이전/다음 달 날짜는 더 어둡게 눌러서 시각적으로 후순위 처리
